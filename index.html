Plexus Board — setup

This app reads and writes your private ops files in the data repository. It needs a GitHub fine-grained Personal Access Token. The app shell is public (GitHub Pages); your token and your data never leave your browser.

1 · Create a fine-grained Personal Access Token

  1. Go to GitHub → Settings → Developer settings → Fine-grained tokens → Generate new token.
  2. Token name: plexus-board (anything).
  3. Resource owner: TGragg89.
  4. Expiration: 90 days (you'll be reminded to renew).
  5. Repository access: "Only select repositories" → choose only plexus-bucket-d.
  6. Permissions → Repository permissions → Contents: Read and write. Leave everything else at "No access".
  7. Generate, then copy the token (starts with github_pat_).

2 · Consent — how your token is stored

  • The token is saved in this browser's localStorage — plaintext at rest in your browser profile on this device.
  • It can touch only the contents of the plexus-bucket-d repository (the access you granted it above). Nothing else.
  • It is sent only to api.github.com, in the request header — never placed in a URL, a commit, or this public repo.
  • To revoke at any time: GitHub → Settings → Developer settings → Fine-grained tokens → Revoke. Then clear it here with "Forget token".
  • Use this only on a private device. Shared/public machines are out of contract.
